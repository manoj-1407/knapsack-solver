Algorithm Analysis

Dynamic Programming — O(n·W)

Builds a 2D table of size (n+1)×(W+1). Each cell dp[i][w] stores the maximum value achievable using items 1..i with capacity w. The recurrence:

dp[i][w] = max(
  dp[i-1][w],
  dp[i-1][w - wi] + vi  // if w ≥ wi
)

Backtracking through the filled table reveals exactly which items were selected. Guaranteed to find the global optimum. Space: O(n·W).

Greedy Heuristic — O(n log n)

Sort items by a chosen criterion, then greedily take items that fit until the knapsack is full. Three strategies are compared:

Ratio Strategy
Sort by v/w descending. Best approximation for most cases. Works perfectly when items can be split (fractional knapsack) but may fail in 0/1 variant.
Value Strategy
Sort by value descending. Maximises immediate gain but may fill capacity with heavy items, missing better combinations.
Weight Strategy
Sort by weight ascending. Fits the most items but completely ignores value — often produces poor results on mixed datasets.

⚠ When Greedy Fails (Counterexample)

Capacity W = 10. Items: A (w=6, v=30, ratio=5.0), B (w=5, v=25, ratio=5.0), C (w=5, v=25, ratio=5.0).
Greedy (ratio) picks A (v=30, fills 6/10). Total = 30.
DP picks B + C (v=25+25=50, fills 10/10). Total = 50.   Greedy is 40% suboptimal here.
